Install LLM-Wiki
Three entry points, one backend: CLI, skill, and MCP.
Host {{ .InstallBaseURL }} Guide {{ .InstallDocURL }}
CLI
Command-line access
Install once, then log in and use the hosted service directly.
For Agents
Read:
{{ .InstallDocURL }}

Then run:
curl -fsSL {{ .InstallScriptURL }} | sh

After install:
llm-wiki system info --base-url {{ .InstallBaseURL }}
llm-wiki auth login --base-url {{ .InstallBaseURL }}
For Humans
curl -fsSL {{ .InstallScriptURL }} | sh
llm-wiki system info --base-url {{ .InstallBaseURL }}
llm-wiki auth login --base-url {{ .InstallBaseURL }}
Skill
Agent-native install
Point the runtime at the official guide or packaged skill.
For Agents
Read and follow:
{{ .InstallDocURL }}

If skill packages are supported, install:
{{ .InstallSkillURL }}
For Humans
open {{ .InstallDocURL }}
curl -O {{ .InstallSkillURL }}
MCP
Protocol access
Use remote MCP or the local stdio bridge with a bearer token.
For Agents
Remote MCP endpoint:
{{ .MCPURL }}

Auth header:
Authorization: Bearer <llm-wiki-token>

Local stdio bridge:
LLM_WIKI_TOKEN=<llm-wiki-token> npx --prefix npm/llm-wiki-mcp llm-wiki-mcp --base-url {{ .InstallBaseURL }}
For Humans
npm install --prefix npm/llm-wiki-mcp --package-lock=false
LLM_WIKI_TOKEN=<llm-wiki-token> npx --prefix npm/llm-wiki-mcp llm-wiki-mcp --base-url {{ .InstallBaseURL }}
