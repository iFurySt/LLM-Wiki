Install Paths

CLI, skill, and MCP. Nothing else on this page.
CLI

Install LLM-Wiki CLI

Use the hosted shell installer to install the local CLI, then run commands against your LLM-Wiki server.
Prompt For AI Agent
Install and use the LLM-Wiki CLI for this workspace.

Read:
{{ .InstallDocURL }}

Then run:
curl -fsSL {{ .InstallScriptURL }} | sh

After install, use:
llm-wiki system info --base-url {{ .InstallBaseURL }} --tenant default
Command For Humans
curl -fsSL {{ .InstallScriptURL }} | sh
llm-wiki system info --base-url {{ .InstallBaseURL }} --tenant default
Skill

Install Skill Package

Give the agent the hosted guide or the official packaged skill so it can follow your LLM-Wiki conventions directly.
Prompt For AI Agent
Read and follow this LLM-Wiki install guide:
{{ .InstallDocURL }}

If your runtime supports skill packages, install:
{{ .InstallSkillURL }}

Use LLM-Wiki as the shared wiki backend for this task.
Command For Humans
open {{ .InstallDocURL }}
curl -O {{ .InstallSkillURL }}
MCP

Connect Through MCP

Use the local MCP bridge or the remote HTTP MCP endpoint so Codex or another client can talk to LLM-Wiki directly.
Prompt For AI Agent
Connect to the local LLM-Wiki MCP server.

Remote MCP endpoint:
{{ .MCPURL }}

Tenant header:
X-LLM-Wiki-Tenant-ID: default

If local stdio MCP is preferred, run:
npx --prefix npm/llm-wiki-mcp llm-wiki-mcp --base-url {{ .InstallBaseURL }} --tenant default
Command For Humans
npm install --prefix npm/llm-wiki-mcp --package-lock=false
npx --prefix npm/llm-wiki-mcp llm-wiki-mcp --base-url {{ .InstallBaseURL }} --tenant default
