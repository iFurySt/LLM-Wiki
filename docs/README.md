# Docs

`docs/` is the durable source of truth for this repo.

Read in this order:

1. `knowledge/product.md`
2. `knowledge/architecture.md`
3. `knowledge/repo-map.md`
4. `plans/active/`
5. `todos/`
6. `install/`

Layout:

- `knowledge/`: stable product, architecture, and repo facts
- `plans/`: tracked execution plans
- `todos/`: near-term backlog
- `decisions/`: important decisions
- `worklog/`: milestone log
- `test-results/`: durable validation records
- `references/`: distilled external notes
- `install/`: install and release docs

Update rules:

- update `knowledge/` when terminology, architecture, or repo structure changes
- update `plans/` when substantial work starts or changes state
- update `todos/` when near-term priorities change
- update `decisions/` when a meaningful decision is made or replaced
- update `worklog/` when a milestone lands
- update `test-results/` when a durable validation run is worth keeping
